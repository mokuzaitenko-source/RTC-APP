Execution Workflow

Simple first. Scale only when metrics prove it.

Phase 1: Define

Set one objective, constraints, and success criteria. Ask for a concrete plan with acceptance checks.

Phase 2: Execute

Implement the smallest testable slice. Keep output practical and measurable.

Phase 3: Measure

Track usable output rate, latency, fallback rate, and next fixes. Evolve only when numbers demand it.
